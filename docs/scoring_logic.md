# Epidauros: スコアリング・ロジック詳細仕様 (Version 1.2)

本ドキュメントでは、AIキャスティング・エージェントにおける俳優の価値評価と検索結果の並び替え（Re-ranking）に用いる、3つの主要なスコアリング・アルゴリズムについて解説します。

---

## 1. 配役の成功度を示す `success_score`
**対象データ:** `Performances` テーブル
**目的:** その配役が、俳優のキャリアにおいていかに「成功した代表作」であるかを 0.0 〜 1.0 で数値化します。

### ステップ 1: 年代別の興行収入正規化
物語のインフレを考慮し、年代（10年単位）ごとに興行収入（`box_office`）の中央値を基準に 0.5 〜 1.0 の間で正規化します。
- 興行収入が不明（0）または不明確な場合は、ベースラインとして **0.5** を付与。
- 年代別の偏差を吸収し、1970年代の名作と2020年代のヒット作を同等に評価します。

### ステップ 2: 配役ランクによる重み付け
作品内でのキャラクターの重要度（`expected_guarantee_rank`）に応じてスコアを加算します。
- **S (主演・トップ)**: 1.0倍
- **A (主要キャスト)**: 0.8倍
- **B (助演・脇役)**: 0.5倍
- **C (端役・カメオ)**: 0.1倍

---

## 2. デモグラフィック・ペナルティ（Soft Filter）
**対象フェーズ:** 検索後の再ランク付け
**目的:** 年齢や性別の不一致を「排除」ではなく「スコア減衰」として扱い、創造的な配役案を浮上させます。

### A. Age Buffer & Penalty (年齢バッファ)
指定されたキャラクターの年齢と、俳優の当時の実年齢（または推定年齢）の差（`age_diff`）を計算します。
- **±5歳以内**: ペナルティなし。
- **±5歳を超える場合**:
    - `DistanceFactor = (abs(age_diff) - 5) / 10`
    - `Score = BaseScore * max(0.2, 1.0 - DistanceFactor)`
    - ※ 離れれば離れるほど評価が下がりますが、下限（0.2倍）を設けることで、セマンティックな一致度が高い場合は「年齢を超えた演技力」として推論されます。

### B. Gender Penalty (性別不一致への寛容)
- **一致する場合**: ペナルティなし (1.0倍)
- **不一致の場合**: 検索スコアを **0.5倍** とします。
- ※ 「不適合」として切り捨てるのではなく、スコアを半減させた上で全体のランキングに再投入します。これにより「意外性のある属性キャスティング」のチャンスが残ります。

---

## 3. ユーザーの意向（Intentions）による調整
キャスティング・ボード上のスイッチにより、検索エンジンの軸を変動させます。

### 🎭 Unexpected (意外性重視)
- **ロジック**: 過去にそのキャラクターのアーキタイプ（例：SHADOW）を演じたことがある俳優を、検索候補から**物理的に除外**、またはスコアを極限まで下げます。
- **結果**: 「普段は悪役をやらない、善人役ばかりのスター俳優を悪役に据える」といった提案が優先されます。

### 🚀 Blockbuster (スターパワー重視)
- **ロジック**: セマンティック（テキスト類似度）と、俳優が持つ累積 `Success Score` を 50:50 の比重で合成します。
- **結果**: 役へのハマり具合だけでなく、世界的な知名度（興行実績）の高い大物スターがランキングの上位に浮上します。

---

## 4. 最終キャスティング・スコアの合成
検索結果の最終的な順位は、以下の重み付けによって決定されます。

`FinalScore = (VectorSimilarity * 0.7 + SuccessScore * 0.3) * DemographicPenalty`

この合成スコアにより、「役柄の意味（意味論）」を第一優先としつつ、「興行実績（成功度）」と「属性（デモグラフィック）」をバランス良く評価した提案が生成されます。
